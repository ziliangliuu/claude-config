# Claude Code 状态栏（用量 Hub）

## 需求

在 Claude Code 底部状态栏实时展示三行信息：

```
Fable 5/hi │ 🟢 1M 12% │ ⏳ 🟢 93% 2h58m │ 📅 🟢 95% 1d4h
📁 project-name
🌿 main★
```

1. **第一行**：模型/effort 档位 │ 上下文窗口已用% │ 5 小时窗口配额 │ 每周配额（有 Sonnet 独立配额时一并显示）。
2. **配额语义**：⏳ 和 📅 显示的是**剩余百分比**（不是已用），后面跟距重置的剩余时间（如 `2h58m`、`1d4h`）。
   颜色灯仍按**已用量**判断：已用 <50% 🟢、<80% 🟡、否则 🔴 —— 剩得越少灯越红。
3. **第二行**：当前工作目录名。**第三行**：git 分支，有未提交改动时加 `★`，非 git 目录显示 `(no git)`。
4. 配额数据优先取 Claude Code 原生 stdin 的 `rate_limits`（无网络请求）；仅当需要 Sonnet 周配额或 stdin 缺字段时才调 Anthropic OAuth 用量 API，结果缓存 5 分钟，缓存过旧时用 ⚠ 替代颜色灯提示数据可能过期。

## 本文件夹内容

| 文件 | 用途 |
|------|------|
| `需求.md` | 本说明 |
| `statusline.sh` | 状态栏脚本本体，安装到 `~/.claude/hooks/statusline.sh` |
| `install.sh` | **一键安装**：查依赖 → 拷脚本 → 合并 `settings.json` → 自测渲染 |

## 依赖

`bash`、`jq`、`curl`（macOS：`brew install jq`；curl 系统自带）。
脚本对 GNU/BSD 的 `date`、`stat` 差异已做兼容，macOS 和 Linux 都能直接跑。

## 安装：macOS / Linux（一键）

```bash
# 在本文件夹目录下执行：
bash install.sh
```

它会：
1. 检查 `jq`、`curl` 是否存在（缺了会提示怎么装并退出）；
2. 把 `statusline.sh` 拷到 `~/.claude/hooks/`；
3. 向 `~/.claude/settings.json` **合并**（不是覆盖）以下字段，并先备份原文件为 `settings.json.bak`：
```json
{
  "statusLine": {
    "type": "command",
    "command": "SHOW_WEEKLY=1 bash ~/.claude/hooks/statusline.sh",
    "padding": 0
  }
}
```
4. 用一份模拟数据自测渲染，直接打印出三行状态栏让你确认。

**新开一个 claude 会话**即可看到状态栏（旧会话不会自动刷新配置）。

## 安装：Windows

脚本是 bash 写的，Windows 下两个选择：
- **WSL 内用 Claude Code**：与 Linux 步骤完全相同；
- 原生 Windows：需要 Git Bash 提供 `bash`，`settings.json` 里 command 写
  `"C:\\Program Files\\Git\\bin\\bash.exe" ~/.claude/hooks/statusline.sh`（路径按实际安装位置调整），
  且需另装 `jq`（`winget install jqlang.jq`）。未在原生 Windows 实测过，遇到问题优先走 WSL。

## 可改参数（环境变量，写在 settings.json 的 command 前缀里）

| 变量 | 默认 | 作用 |
|------|------|------|
| `SHOW_WEEKLY` | `0` | 设为 `1` 显示每周配额 + Sonnet 周配额（推荐开，安装脚本默认开） |
| `REFRESH_INTERVAL` | `300` | 用量 API 缓存秒数；`0` = 每次渲染都请求（有限流风险，别设） |
| `TIMEZONE` | 系统默认 | 如 `America/New_York` |
| `USAGE_FILE` | `~/.claude/usage-exact.json` | 用量缓存文件位置 |

## 验证

```bash
# 喂模拟数据，应输出三行状态栏，⏳ 显示 93%、📅 显示 95%（剩余语义）：
NOW=$(date +%s); printf '{"model":{"display_name":"Opus 4.8"},"context_window":{"used_percentage":12,"context_window_size":1000000},"workspace":{"current_dir":"'"$HOME"'"},"rate_limits":{"five_hour":{"used_percentage":7,"resets_at":'$((NOW+10680))'},"seven_day":{"used_percentage":5,"resets_at":'$((NOW+100800))'}}}' \
  | SHOW_WEEKLY=1 bash ~/.claude/hooks/statusline.sh
```

真机验证：新开 claude 会话，底部应出现状态栏；发几条消息后 ⏳ 的剩余百分比会随用量下降。

## 已知的坑

- **合并而非覆盖 settings.json**：`settings.json` 里通常还有 hooks、model 等字段，手工装时务必用 `jq '.statusLine = ...'` 合并；`install.sh` 已处理并自动备份。
- **剩余 vs 已用**：上下文那一项（`1M 12%` / `Ctx 12%`）显示的是**已用**，⏳/📅 显示的是**剩余**——这是刻意设计，别"顺手统一"。
- **Sonnet 周配额只来自 API**（stdin 没有该字段），所以开 `SHOW_WEEKLY=1` 后脚本会低频调用 OAuth 用量 API，需要 `~/.claude/.credentials.json` 存在（正常登录过 Claude Code 就有）。
- 状态栏配置只在**新会话**生效；改完脚本或 settings 后要重开会话看效果。
- `resets_at` 时间戳格式两种都要兼容：stdin 给的是 epoch 秒，API 给的是 ISO 8601（脚本已处理，改脚本时别弄丢 `iso_to_epoch` 的 BSD/GNU 双路径）。
