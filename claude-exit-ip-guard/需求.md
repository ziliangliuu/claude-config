# Claude Code 启动出口 IP 校验

## 需求

1. **启动时**：在终端输入 `claude` 启动 Claude Code 时，检查当前出口 IP 是否为期望值（默认 `198.65.8.45`，存于 `~/.claude/hooks/expected-exit-ip`）。
   不匹配（或取不到 IP，视为网络异常）就提示网络有问题并**阻止启动**；无论通过还是拦截都给出明确提示。
2. **运行中**：一个窗口可能开着好多天、中途网络切换多次，VPN 可能已断而 claude 仍在跑。
   因此还要在**每次发消息前**再校验一次出口 IP，VPN 不对就**拦下这条消息**，恢复网络后才能继续。

> 两层防护：启动时用 shell 函数挡一次（下方「方案原理」），运行中用 `UserPromptSubmit` 钩子每次提交再挡一次（下方「第二层」）。

## 本文件夹内容

| 文件 | 用途 |
|------|------|
| `需求.md` | 本说明 |
| `install.sh` | **一键安装** · macOS / Linux，幂等（先清旧块再装两层），推荐直接用它 |

| `check-exit-ip-prompt.sh` | **运行中层** · macOS / Linux 的 UserPromptSubmit 钩子脚本 |
| `check-exit-ip-prompt.ps1` | **运行中层** · Windows 的 UserPromptSubmit 钩子脚本 |

> 整个文件夹可直接拷贝到别的电脑，按下面对应系统的步骤安装。

## 方案原理

用一个与命令**同名的 shell 函数 `claude`** 包住真正的命令：
- 敲 `claude` 时优先命中函数 → 先查出口 IP。
- IP 不符 → 直接返回，**根本不启动 claude 进程**（真正的「不能启动」）。
- IP 相符 → 调用真正的 claude 可执行文件启动。

> ⚠️ 为什么不用 Claude Code 的 `SessionStart` 钩子？
> 实测钩子的 `{"continue": false}` **拦不住启动**——它在 claude 进程内部运行，太晚了。
> 必须在 shell 层、进程启动前拦截，所以用函数包装。

## 安装：macOS / Linux（推荐一键脚本）

```bash
# 在本文件夹目录下执行，幂等，可重复运行：
bash install.sh
```
它会一次做完：写入默认期望 IP（`198.65.8.45`，已存在则沿用）→ 多源探测当前出口 IP 并对比 →
装运行中层钩子并合并 `settings.json` → 把启动层函数装进当前 shell 的 rc（**先清掉旧块再写，自动去重**）。
然后**新开一个终端窗口**使启动层生效，**新开 claude 会话**使运行中钩子生效。

> 环境清理：脚本靠 `# === … BEGIN/END ===` 标记定位旧块并整段删除后重写，所以重复运行、
> 或机器上已有旧版本，都不会残留重复的 `claude()` 函数。手动 `cat >>` 没有这个去重，别混用。

```bash
cat claude-guard.sh >> ~/.bash_profile
```

## 安装：Windows（PowerShell，推荐一键脚本）

```powershell
# 在本文件夹目录下执行，幂等，可重复运行：
```

然后**新开一个 PowerShell 窗口**使启动层生效，**新开 claude 会话**使运行中钩子生效。

## 期望 IP 单点配置（重要）

期望出口 IP **只存一个文件**：`~/.claude/hooks/expected-exit-ip`（Windows：`%USERPROFILE%\.claude\hooks\expected-exit-ip`），
内容就是一行 IP。**启动层和运行中层两个脚本都读它**，所以换 IP 只改这一个文件、改一次即可。

```bash
# 首次配置 / 换 IP：
echo "198.65.8.45" > ~/.claude/hooks/expected-exit-ip
```
- 仓库里的 `expected-exit-ip.example`（默认 `198.65.8.45`）是模板，拷成 `~/.claude/hooks/expected-exit-ip` 并按需改成真实 IP。
- 该文件缺失或为空时，两层都会**拦截**（fail-closed），并提示先配置。

## 其他可改参数

| 想改什么 | macOS/Linux (`.sh`) | Windows (`.ps1`) |
|---------|--------------------|------------------|
| 期望出口 IP | 改 `~/.claude/hooks/expected-exit-ip` 一个文件（见上） | 同左 |
| IP 检测服务 | 脚本内 `IP_SERVICES` 数组（多源，见下） | 脚本内 `$ipServices` 数组（多源，见下） |
| 单服务超时秒数 | `.sh` 启动层 `--max-time 4`、运行层 `--max-time 3` | `.ps1` HttpClient `Timeout` 4s |

### 出口 IP 检测：多源 + 并发 + 命中即放行（重要）

检测当前出口 IP **不依赖单一网站**，而是**并发**请求一组 IP 回显服务。**谁先返回、且结果
等于期望 IP，立即放行**，不等其余服务——这是「通过」路径的最坏耗时，通常远小于单个超时。
只有【全程没有任何服务命中期望 IP】时（可能确实被拦截，也可能是探测服务集体抽风），才会
等全部完成/超时，再按优先级取【第一个返回合法 IPv4（逐段 0-255 校验）】的结果用于日志与
拦截提示；这条「未命中」路径的最坏耗时才是**单个超时**（而非各服务累加）。这个不对称是刻意
设计：「放行」判定越快越好，但「拦截」判定要更谨慎，不能因为某个服务慢/挂了、而正确的服务
还没返回，就提前误判拦截。默认服务列表（四个脚本一致）：

```
https://api.ipify.org
https://ifconfig.me/ip
https://icanhazip.com
https://ipinfo.io/ip
https://checkip.amazonaws.com
https://api.ip.sb/ip
```

- 实现：`.sh` 用后台 `curl &` + 轮询完成标记、命中即 `kill`/`disown` 掉尚未返回的请求后立即退出；
  `.ps1` 用 `HttpClient` 异步 `GetStringAsync` + `Task.WaitAny` 循环等待，命中即跳出。
  `claude-guard.sh` 会被 zsh/bash 直接 `source`（无 shebang），两种 shell 数组下标起始不同
  （zsh 从 1、bash 从 0），故其实现刻意不对服务数组做数字下标访问，只用顺序遍历 + 独立计数器
  对应临时文件，两种 shell 下行为一致；`check-exit-ip-prompt.sh` 由 Claude Code 固定以
  `bash script.sh` 调用，不受此限制，可直接用数组下标。
- 增删/改序：编辑脚本内的 `IP_SERVICES`（`.sh`）或 `$ipServices`（`.ps1`）数组。
- 命中的服务记进日志（`source=[...]` 列，命中即放行时额外带 `short_circuit=yes`），全失败时为 `source=[NONE]`。
- ⚠️ **钩子超时是 fail-open（放行）**：Claude Code 的 `UserPromptSubmit` 钩子若**超时或以非
  0/2 退出码异常结束**，官方语义是**取消钩子、放行本次提交**，并非拦截。所以本方案的
  fail-closed 只覆盖「探测到 IP 但不符」「配置缺失(正常退出+block)」；**脚本本身报错/超时那一档是放行**。
  命中即放行进一步把「通过」路径的耗时压到最快命中那个请求的量级，「拦截」路径仍有单个超时
  兜底、`timeout` 设 20s 留足余量，两者都是为尽量避免撞进这个放行窗口。

## 验证

**macOS/Linux：**
```bash
zsh -c 'source ~/.zshrc >/dev/null 2>&1; type claude'   # 应显示 claude is a shell function
```
把 `~/.claude/hooks/expected-exit-ip` 里的 IP 临时改成 `1.2.3.4` → 新开终端敲 `claude` → 应看到红色拦截、claude 不启动 → 测完改回。

**Windows：**
```powershell
Get-Command claude                # 应显示 CommandType = Function
```
把 `%USERPROFILE%\.claude\hooks\expected-exit-ip` 里的 IP 临时改成 `1.2.3.4` → 新开 PowerShell 敲 `claude` → 应看到拦截、claude 不启动 → 测完改回。

## 第二层：运行中检查（UserPromptSubmit 钩子）

启动层的 shell 函数只在**敲 `claude` 那一刻**查一次。若窗口开着好几天、中途网络切换、VPN 断了，
已经在跑的 claude 不会再查——存在盲区。用 `UserPromptSubmit` 钩子补上：**每次你提交消息前**再查一次，
VPN 不对就拦下这条消息。

> 注意区分：启动拦截**不能**用 `SessionStart` 钩子（`continue:false` 拦不住，见上文）；
> 但运行中拦截**要**用 `UserPromptSubmit` 钩子——它的 `{"decision":"block"}` 能真正拦住这次提交。二者用途不同。

> macOS / Linux 用了 `install.sh` 的话，这一层已经装好，无需重复操作。下面是手动/Windows 步骤。

### 安装：macOS / Linux（手动）
1. 把 `check-exit-ip-prompt.sh` 拷到 `~/.claude/hooks/`。**脚本本身不含 IP**——期望 IP 统一从
   `~/.claude/hooks/expected-exit-ip` 读取（见上「期望 IP 单点配置」），所以不需要改脚本内容。
2. 在 `~/.claude/settings.json` 合并进以下 hook（保留已有字段，别整体覆盖）：
```json
{
  "hooks": {
    "UserPromptSubmit": [
      { "hooks": [ { "type": "command", "command": "bash ~/.claude/hooks/check-exit-ip-prompt.sh", "timeout": 20 } ] }
    ]
  }
}
```

### 安装：Windows
1. 把 `check-exit-ip-prompt.ps1` 拷到 `%USERPROFILE%\.claude\hooks\`。**脚本本身不含 IP**，期望 IP 同样从 `expected-exit-ip` 文件读取，无需改脚本。
2. 在 `settings.json` 合并进：
```json
{
  "hooks": {
    "UserPromptSubmit": [
      { "hooks": [ { "type": "command", "shell": "powershell", "command": "powershell -NoProfile -ExecutionPolicy Bypass -File \"$env:USERPROFILE\\.claude\\hooks\\check-exit-ip-prompt.ps1\"", "timeout": 20 } ] }
    ]
  }
}
```

### 验证
- 脚本自测（改错期望 IP 应输出 block JSON）：
  `echo '{}' | bash ~/.claude/hooks/check-exit-ip-prompt.sh` → 正常网络下应**无输出**（放行）。
- 真机验证：把 `~/.claude/hooks/expected-exit-ip` 里的 IP 临时改成 `1.2.3.4` → **新开一个 claude 会话** → 发任意一条消息 → 应被拦下并提示；测完改回。
- 钩子在**新会话**才生效；已开着的会话需重开或在 `/hooks` 里重载配置。

## 注意事项

- **必须新开终端/窗口**：配置只在新 shell 启动时加载，旧窗口不受影响。
- 运行中钩子在**每次发消息前**多一次联网查询（命中首个可用服务通常 <1s）；网络异常时会依次
  尝试服务列表，最坏情况约等于「失败服务数 × 单服务超时」，全失败后再拦截。
- 每次启动会多一次联网查询（约 <1s）；断网时会等满各服务超时后按「网络异常」拦截。
- 临时绕过校验直接启动：
  - macOS/Linux：`command claude` 或绝对路径 `~/.npm-global/bin/claude`
  - Windows：`& (Get-Command claude.cmd -CommandType Application).Source`
- 运行日志：`~/.claude/hooks/check-exit-ip.log`（Windows 为 `%USERPROFILE%\.claude\hooks\check-exit-ip.log`）。
- **边界（固有局限）**：本方案是「自我约束」级别，不是强制安全隔离。非交互 shell、脚本、IDE/GUI 若直接调用 claude 可执行文件（如 `~/.npm-global/bin/claude`）会绕过本函数。若需强制，须在网络层/系统层做限制。
- Windows 提示用 `OK` / `X` 前缀而非 emoji，避免旧版 PowerShell 控制台乱码。
