# Claude Code 启动出口 IP 校验

## 需求

在终端输入 `claude` 启动 Claude Code 时，检查当前出口 IP 是否为 `YOUR_EXIT_IP`。
不匹配（或取不到 IP，视为网络异常）就提示网络有问题并**阻止启动**；无论通过还是拦截都给出明确提示。

## 本文件夹内容

| 文件 | 用途 |
|------|------|
| `需求.md` | 本说明 |
| `claude-guard.sh` | **macOS / Linux**（zsh 或 bash）版函数，追加到 `~/.zshrc` 或 `~/.bashrc` |
| `claude-guard.ps1` | **Windows**（PowerShell）版函数，追加到 `$PROFILE` |

> 整个文件夹可直接拷贝到别的电脑，按下面对应系统的步骤安装。

## 方案原理

用一个与命令**同名的 shell 函数 `claude`** 包住真正的命令：
- 敲 `claude` 时优先命中函数 → 先查出口 IP。
- IP 不符 → 直接返回，**根本不启动 claude 进程**（真正的「不能启动」）。
- IP 相符 → 调用真正的 claude 可执行文件启动。

> ⚠️ 为什么不用 Claude Code 的 `SessionStart` 钩子？
> 实测钩子的 `{"continue": false}` **拦不住启动**——它在 claude 进程内部运行，太晚了。
> 必须在 shell 层、进程启动前拦截，所以用函数包装。

## 安装：macOS / Linux

```bash
# zsh（macOS 默认）：
cat claude-guard.sh >> ~/.zshrc
# bash（Linux）：
cat claude-guard.sh >> ~/.bashrc
# bash（macOS 登录 shell 默认读 .bash_profile，不读 .bashrc）：
cat claude-guard.sh >> ~/.bash_profile
```
然后**新开一个终端窗口**即可生效。

## 安装：Windows（PowerShell）

```powershell
# 1) 确保配置文件存在
New-Item -ItemType File -Force -Path $PROFILE
# 2) 把脚本内容追加进去（在本文件夹目录下执行，务必带 -Encoding UTF8，否则 PS 5.1 下中文会乱码）
Get-Content -Raw -Encoding UTF8 .\claude-guard.ps1 | Add-Content -Encoding UTF8 $PROFILE
# 3) 若加载被安全策略拦住，执行一次：
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```
然后**新开一个 PowerShell 窗口**即可生效。

- 若用的是 CMD 而非 PowerShell：CMD 没有等价的函数机制，建议改用 PowerShell；
  或把校验写成一个 `claude.cmd` 批处理放在 PATH 靠前的目录里（需另行编写）。

## 关键参数（改这里）

| 想改什么 | macOS/Linux (`claude-guard.sh`) | Windows (`claude-guard.ps1`) |
|---------|-------------------------------|------------------------------|
| 期望出口 IP | `expected_ip="YOUR_EXIT_IP"` | `$expectedIp = "YOUR_EXIT_IP"` |
| IP 检测服务 | `https://api.ipify.org` | 同左 |
| 超时秒数 | `--max-time 8` | `-TimeoutSec 8` |

## 验证

**macOS/Linux：**
```bash
zsh -c 'source ~/.zshrc >/dev/null 2>&1; type claude'   # 应显示 claude is a shell function
```
把 `expected_ip` 临时改成 `1.2.3.4` → 新开终端敲 `claude` → 应看到红色拦截、claude 不启动 → 测完改回。

**Windows：**
```powershell
Get-Command claude                # 应显示 CommandType = Function
```
把 `$expectedIp` 临时改成 `1.2.3.4` → 新开 PowerShell 敲 `claude` → 应看到拦截、claude 不启动 → 测完改回。

## 注意事项

- **必须新开终端/窗口**：配置只在新 shell 启动时加载，旧窗口不受影响。
- 每次启动会多一次联网查询（约 <1s）；断网时会等满超时后按「网络异常」拦截。
- 临时绕过校验直接启动：
  - macOS/Linux：`command claude` 或绝对路径 `~/.npm-global/bin/claude`
  - Windows：`& (Get-Command claude.cmd -CommandType Application).Source`
- 运行日志：`~/.claude/hooks/check-exit-ip.log`（Windows 为 `%USERPROFILE%\.claude\hooks\check-exit-ip.log`）。
- **边界（固有局限）**：本方案是「自我约束」级别，不是强制安全隔离。非交互 shell、脚本、IDE/GUI 若直接调用 claude 可执行文件（如 `~/.npm-global/bin/claude`）会绕过本函数。若需强制，须在网络层/系统层做限制。
- Windows 提示用 `OK` / `X` 前缀而非 emoji，避免旧版 PowerShell 控制台乱码。
